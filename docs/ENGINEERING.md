## Code Quality

**Linter:** [Ruff](https://docs.astral.sh/ruff/) â€” configured identically across all 4 Python services (`backend`, `bot`, `scraper`, `core`).

**Shared config** (in each `pyproject.toml`):

```toml
[tool.ruff]
target-version = "py312"
line-length = 100

[tool.ruff.lint]
select = ["B", "E", "F", "I", "N", "RET", "RUF", "S", "SIM", "TCH", "UP"]
```

**Rule set:**

| Rule | Category | What it catches |
| ---- | -------- | --------------- |
| B | flake8-bugbear | Mutable default args, missing `raise from`, `zip()` without `strict=` |
| E | pycodestyle | Whitespace, indentation, line length |
| F | pyflakes | Unused imports, undefined names |
| I | isort | Import ordering (with `known-first-party` per service) |
| N | PEP 8 naming | Lowercase variables in functions, class naming conventions |
| RET | flake8-return | Unnecessary assignments before `return` |
| S | flake8-bandit | Hardcoded secrets, `assert` in production, unsafe XML parsing, `0.0.0.0` binding |
| SIM | flake8-simplify | `try/except/pass` â†’ `contextlib.suppress()`, collapsible `if` blocks |
| RUF | ruff-specific | Unused `noqa` directives, ambiguous Unicode, mutable class defaults |
| TCH | flake8-type-checking | Moves type-only imports behind `TYPE_CHECKING` (faster runtime imports) |
| UP | pyupgrade | Modern Python syntax (union types, f-strings, etc.) |

**Per-file ignores** (service-specific, justified):

- `tests/*` â†’ `S101` (assert), `S105` (test secrets) â€” standard for test files
- `api/*`, `auth.py` â†’ `B008` â€” FastAPI `Depends()` in signatures is the framework convention
- `__main__.py` â†’ `S104` â€” `0.0.0.0` binding is intentional inside Docker containers
- `schemas/*` â†’ `S105` â€” Pydantic default values like `token_type = "bearer"` are not secrets
- `alembic/versions/*` â†’ `F821` â€” autogenerated migrations reference forward-declared names
- `scraper/{products,sitemap}.py` â†’ `S314` â€” `lxml`/`ElementTree` XML parsing on trusted SAQ data

**Frontend:** ESLint + Prettier. TypeScript strict mode.

**Formatting:** `ruff format` (Python), `prettier` (frontend).

**Commands:** `make lint` (check all), `make format` (fix all). Pre-commit hook runs `make lint`; pre-push runs full lint + test.

## Observability

Structured logging via [Loguru](https://github.com/Delgan/loguru) across all Python services. Each service tags logs with service name, log level, and ISO timestamps.

| Service | What's logged |
|---------|---------------|
| Backend | API request lifecycle, Claude API calls (intent, curation, sommelier), recommendation pipeline latency breakdown |
| Scraper | Per-product success/error, run summary at exit (inserted/updated/errors/restocked/delisted/duration) |
| Bot | Command usage, notification delivery, error handler output |

Scraper uses named exit codes (`EXIT_OK`/`EXIT_PARTIAL`/`EXIT_FATAL`) for systemd integration. Loguru `SUCCESS` level marks per-product completions.

Recommendation pipeline logs each request to the `recommendation_logs` table (query, intent, returned SKUs, latency breakdown by stage).

Docker log rotation configured per service (10MB Ă— 3 files = 30MB cap). Platform-level monitoring (Uptime Kuma, Umami) is managed in the [infra repo](https://github.com/vpatrin/infra).

## Platform

**Build & CI:**

- Makefile with per-service and combined targets (install, dev, lint, format, test, coverage, audit, build, run)
- CI pipeline: lint + test + audit per Python service, Hadolint for Dockerfiles, coverage enforcement, concurrency cancel-in-progress
- Frontend CI: ESLint, TypeScript typecheck, production build, bundle size tracking
- Trivy container scan on built images (PR checks + tag push)
- Scoped git hooks: pre-commit lints changed services, pre-push tests changed services + checks lock files

**CD pipeline:**

- Build + Trivy + push to GHCR on tag push
- `docker-compose.prod.yml` â€” GHCR images, restart policies, no dev volumes
- Automated deploy on tag push via SSH â€” see [PRODUCTION.md](PRODUCTION.md)
- Production secrets encrypted with sops + age, decrypted at deploy time
- GitHub Release auto-created with changelog on tag push
- Production environment with main-only deployment branch policy and tag protection

**Docker:**

- Multi-stage builds, non-root users, read-only filesystem, `no-new-privileges`, memory limits
- Dev: `docker compose up` (postgres only, services run bare-metal for hot reload)
- Prod: full compose with GHCR images

See [PRODUCTION.md](PRODUCTION.md) for deploy process. Systemd timers and backup scheduling are documented in the [infra SERVICE_CATALOG.md](https://github.com/vpatrin/infra/blob/main/docs/SERVICE_CATALOG.md).

## SRE

**Resilience patterns:**

- Scraper: idempotent (safe to re-run), systemd `Persistent=true` (runs on reboot if missed)
- Bot: 24/7 via PTB's built-in error handler with Loguru forwarding
- Backend: `/health` endpoint for container health checks and Uptime Kuma
- All Claude/OpenAI calls: graceful fallback on API errors (returns degraded response, never crashes)

**Environment segregation:**

- Single `.env` pattern with env-specific defaults
- Pydantic `BaseSettings` with fail-fast on missing required vars
- `ENVIRONMENT` flag gates debug behavior
- `DATABASE_ECHO` flag for SQL query logging in dev
- `docker-compose.prod.yml` separates prod config from dev
